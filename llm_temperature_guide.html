Section 1 — Definition

What is Temperature in LLMs?

The simple explanation

Temperature controls how random or creative an LLM's output is. Think of it like a confidence dial:

  • Low temperature (close to 0) → The model picks the most likely word every time. Predictable, focused, deterministic.
  • High temperature (close to 2) → The model gives more equal chances to unlikely words too. More creative, surprising, but sometimes incoherent.
Real-world analogy: Imagine asking 10 people "What's a fruit?" Low temperature = everyone says "Apple". High temperature = you get Apple, Durian, Starfruit, Dragon fruit, and maybe "Tuesday" from one person.

The technical explanation

Before the model picks the next word (token), it computes a score (called a logit) for every word in its vocabulary. Temperature scales those scores before converting them to probabilities:

P(tokeni) = e(scorei / T) / Σ e(scorej / T)

Where T is the temperature. As T → 0, the top token gets nearly 100% probability (greedy). As T → ∞, all tokens become equally likely (pure random).

Section 2 — Edge Cases

Important Edge Cases to Know

Edge Case 1 — Common Misconception

❌ "Higher temperature always helps with complex reasoning"

It's tempting to think that if a model is "stuck", cranking up the temperature will help it reason better. Actually, for tasks like machine translation and summarization, higher temperature consistently hurts performance — sometimes by up to 192%. These tasks are deterministic: there's one right answer, and randomness only introduces errors.

Edge Case 2 — The Mutation Temperature

⚠️ Performance doesn't degrade gradually — it falls off a cliff

Models don't smoothly get worse as temperature rises. Instead, there's a "mutation temperature" — a sudden sharp drop. For small models, this cliff appears around T=1.0–1.3. For large models, it's around T=1.6–1.9. This is especially dangerous for instruction-following tasks — the model seems fine and then suddenly starts ignoring instructions.

Edge Case 3 — The Temperature Paradox

🔀 Complex tasks mix abilities, so temperature effects cancel out

When you evaluate a model on a task that requires both logic and creativity (like advanced math story problems), high temperature helps creativity but hurts reasoning. These effects can cancel each other out, making it look like temperature doesn't matter — when it actually matters a lot for each individual capability underneath.

Edge Case 4 — Small vs Large Models Behave Differently

⚠️ The same temperature can have opposite effects by model size

A temperature of 1.3 might improve creativity for a 70B model but destroy it for a 1B model. Small models hit their "creativity peak" earlier and fall off faster. Never assume one temperature setting works across model sizes. Always test per model.

Edge Case 5 — T=0 Is Not Always Best for Accuracy

✅ Zero temperature isn't always optimal even for logical tasks

For causal reasoning, medium and large models actually show a slight performance improvement at T=1.3 compared to T=0. The hypothesis is that for counterintuitive problems, a small amount of randomness helps the model explore reasoning paths it wouldn't take greedily. So "always use T=0 for logic" is a myth.

Edge Case 6 — Quantization Doesn't Change Temperature Behavior

❌ "4-bit quantization changes optimal temperature settings"

Research shows that FP16 and 4-bit quantized models have the same optimal temperature ranges — the behavior is consistent across precision levels. The absolute performance may differ by 10–20%, but the shape of the temperature-performance curve stays the same. So you can safely tune temperature on a quantized model and apply those settings to a full-precision model.

Edge Case 7 — Temperature Interacts With Other Parameters

🔀 Top-P and repetition penalty can shift temperature's effect

Temperature is not the only knob. For creative tasks, Top-P has an even larger effect than temperature. Changing Top-P from 0.8 to 1.0 while keeping temperature fixed can cause negative performance correlation for creativity tasks. When tuning temperature, keep Top-P and repetition penalty fixed first, or you'll be chasing confounded results.

Section 3 — Interview Prep

Interview Questions & Answers

Click any question to expand the answer.

Q1 What is sampling temperature in an LLM, and how does it work mathematically? ▾

Temperature is a hyperparameter that scales the model's output logits before the softmax function converts them into probabilities. Formally, the probability of token i is:

P(i) = exp(logit_i / T) / sum(exp(logit_j / T)) for all j in vocabulary

When T is very small (approaching 0), the exponentials amplify differences — the highest-scoring token gets almost all the probability mass, making the model deterministic. When T is large (approaching infinity), differences get compressed and all tokens become nearly equally likely, producing random output.

Think of temperature like a volume knob on "surprise". Turn it down and the model says the obvious thing. Turn it up and it might say something unexpected — for better or worse.
Q2 What's the difference between temperature = 0 and temperature = 1? ▾

Temperature = 0 (greedy decoding): The model always picks the single most probable next token. The output is completely deterministic — running the same prompt twice gives the exact same result. No creativity, maximum focus.

Temperature = 1 (default sampling): The model samples from the probability distribution as-is, without modification. Each run may produce a slightly different output. Most APIs use this as default because it balances coherence and variety.

In practice, T=0 is preferred for code, math, and factual answers. T=1 is a good default for conversational or open-ended tasks where variety is acceptable.
Q3 Does higher temperature always improve creativity? What's the research finding? ▾

Not always. The relationship follows an inverted-U curve: creativity first increases with temperature, then decreases. The optimal temperature for creativity is around T=1.3 for medium and large models.

Small models are different — they show a marked creativity drop as early as T=1.0. This is because small models have weaker internal representations and can't "channel" randomness productively; they just produce incoherent text instead of creative text.

It's like adding spice to food. A little makes it more interesting. Too much and nobody can taste anything else. And a beginner cook (small model) gets overwhelmed at a much lower spice level than an expert.
Q4 What is the "mutation temperature" and why should developers care? ▾

The mutation temperature is the point at which a model's performance drops sharply — not gradually — as temperature increases. It's the "cliff edge" of the temperature curve.

  • Small models (1B–4B): mutation around T = 1.0–1.3
  • Medium models (7B–13B): mutation around T = 1.3–1.6
  • Large models (40B–80B): mutation around T = 1.6–1.9, or above 2.0

Developers care because if you set T=1.5 thinking it's "a bit creative", a small model may have already crossed its cliff and will give incoherent or instruction-ignoring output. Always know your model size when choosing temperature.

Q5 Why does machine translation degrade the most with high temperature compared to other tasks? ▾

Translation is inherently deterministic — there is a specific correct rendering of a sentence in another language. Unlike creative writing where many outputs can be "correct", a mistranslated word is simply wrong.

Research found that temperature can cause up to 192% relative performance variation in machine translation — the highest of all six abilities tested. This happens because:

  • Higher temperature increases the probability of rare or incorrect words
  • Rare words in a translation context are usually wrong words, not creative choices
  • Even small token-level errors compound into broken sentences
For any translation or structured output task, always use low temperature (T ≈ 0.1). Treat it like code generation — there's a right answer and randomness only introduces bugs.
Q6 What is the "Temperature Paradox" and how does it affect benchmarking? ▾

The Temperature Paradox occurs when a benchmark mixes tasks that need opposing temperature settings. For example, a complex math word problem requires:

  • Low temperature for logical precision
  • Higher temperature for creative problem-solving approaches

These effects cancel each other out, making it appear that temperature has little impact. But if you isolate each ability and test them separately, you find temperature has a dramatic effect on each one individually.

This is why general benchmarks like AGI-Eval often show no temperature effect — the signals cancel. The lesson is to always benchmark temperature effects on isolated, capability-specific tasks.

It's like measuring whether sunscreen works by applying it to half your body and spending the day half in sun, half in shade. The "average" result would mislead you.
Q7 Why might you use a BERT-based temperature selector in a production system? ▾

Different prompts require different optimal temperatures. A system that always uses T=1.0 will be suboptimal: too random for translation prompts, too conservative for creative prompts.

A BERT-based temperature selector classifies each incoming prompt by its primary required ability (translation, creativity, reasoning, etc.) and then routes it to the pre-computed optimal temperature for that ability and model size.

Research showed this approach improved SuperGLUE benchmark performance meaningfully for small (1B) and medium (8B) models — particularly for harder tasks like word sense disambiguation (WIC). For large models, the gains were minimal since large models are more robust to temperature changes.

This approach is valuable in resource-constrained scenarios. When you can't afford a large model, squeezing extra performance from a small model via optimal temperature selection is a practical alternative.
Q8 How does temperature interact with Top-P and Top-K sampling? ▾

These are three separate ways to shape the output distribution:

  • Temperature — reshapes the entire probability distribution by scaling logits before softmax
  • Top-K — restricts sampling to only the K most probable tokens (hard cutoff)
  • Top-P (Nucleus sampling) — restricts sampling to the smallest set of tokens whose cumulative probability exceeds P (dynamic cutoff)

For most tasks, changing Top-K or Top-P doesn't significantly change how temperature affects performance. But for creative tasks, Top-P has a large independent effect — high Top-P (close to 1.0) with high temperature can cause negative performance correlation even at moderate temperatures.

Best practice: fix Top-P = 0.9 and repetition penalty = 1.0 first, then tune temperature. This isolates temperature's effect and avoids confounded results.
Q9 You're building a RAG (Retrieval-Augmented Generation) chatbot. What temperature would you choose and why? ▾

For a RAG system, the primary requirements are accuracy (don't hallucinate), faithfulness to retrieved documents, and natural-sounding responses. This suggests a low-to-moderate temperature.

Recommended approach:

  • For the retrieval step and any structured output: T = 0.1–0.3
  • For generating the final conversational response: T = 0.5–0.7
  • Keep it below T=1.0 — above this, instruction-following degrades and the model may start ignoring the retrieved context
Think of it like a lawyer citing a document in court. You want them to accurately represent what the document says (low temperature for faithfulness) but still speak naturally to the judge (moderate temperature for tone).
Q10 Does 4-bit quantization change how you should set temperature? ▾

No — the shape of the temperature-performance curve is essentially the same in FP16 and 4-bit quantized models. The optimal temperature ranges for each task type remain consistent across precision levels.

The main difference is absolute performance level: quantized models may score 10–20% lower overall, but their temperature sensitivity pattern is the same. If T=0.7 is optimal for summarization in FP16, it's also optimal in 4-bit.

This is practically important because it means you can:

  • Tune temperature on a cheaper quantized model
  • Apply those settings to a full-precision deployment
  • Save significant compute costs during hyperparameter search
Q11 A user complains the LLM is giving "boring and repetitive" answers. What do you investigate first? ▾

First, check the temperature setting. If it's below 0.5, the model is likely in "safe mode" — always picking the most probable token, which naturally feels repetitive since the most probable words are always common, generic words.

Immediate steps:

  • Try increasing temperature to 0.8–1.0 for general conversation
  • Check repetition penalty — if it's set too high (>1.5), the model avoids repeating any word even when repetition would be natural
  • Check Top-P — if it's very low (0.5), only a tiny set of tokens can be selected, limiting vocabulary variety
  • Check if the system prompt is too restrictive, causing the model to "stay safe" linguistically
Usually, nudging temperature from 0.3 to 0.7–0.9 is the fastest fix for "boring output" complaints without risking incoherence.
Q12 Why does temperature matter less for large models than for small models? ▾

Large models have richer internal representations and stronger priors learned from more parameters and data. This makes them more "robust" — their top token probabilities are more confident and well-calibrated, so scaling the logits by temperature has less dramatic effect on which tokens end up getting selected.

Concretely from research: for large models (40B–80B), the coefficient of variation (performance variation across temperatures) is much smaller across all six abilities. They also have higher mutation temperatures, meaning they can tolerate more randomness before performance collapses.

For small models (1B–4B), the probability distributions are flatter and less confident — so a small change in temperature has an outsized effect. Small models are much more sensitive and require more careful temperature tuning.
